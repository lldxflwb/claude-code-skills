# TikZ Skill 设计计划

> 日期: 2026-03-16
> 状态: 已确认

## 1. 概述

创建一个 Claude Code skill，让 AI 能编写 TikZ 代码、编译为 SVG，并通过内置 HTTP 服务返回可点击的预览链接。

**用户体验流程：**
```
用户: "画一个神经网络架构图"
  → Claude 编写 TikZ 代码
  → 调用 render_tikz.py 编译为 SVG
  → 返回 http://10.8.0.2:9094/renders/neural_network.svg
用户在浏览器中查看矢量图
```

## 2. 架构设计

### 2.1 目录结构

```
skills/tikz/
├── SKILL.md                # 技能指令（Claude 读取）
├── setup.md                # 依赖安装引导
└── scripts/
    ├── render_tikz.py      # TikZ → PDF → SVG 渲染脚本
    ├── tikz_server.py      # HTTP 展示服务
    └── ensure_server.py    # 检测/启动 server 的入口脚本
```

**运行时目录（不在 skill 目录中）：**
```
~/.cache/tikz-skill/
├── renders/                # SVG/PDF 产物
├── server.json             # 状态文件 {pid, host, port}
└── server.log              # server 日志
```

### 2.2 渲染层

**工具链：** `tectonic`（优先） / `pdflatex`（fallback）→ PDF → SVG（`pdf2svg`）

**render_tikz.py 职责：**
1. 接收参数：TikZ 代码（stdin 或文件路径）、输出文件名
2. 调用 `ensure_server.py` 确保 server 运行
3. 生成临时 `.tex` 文件（包裹 `\documentclass{standalone}` + TikZ 代码）
4. 调用 tectonic/pdflatex 编译为 PDF
5. 调用 pdf2svg 转为 SVG
6. 将 SVG 放入 `~/.cache/tikz-skill/renders/`
7. 输出可访问的 URL：`http://{TIKZ_SERVER_HOST}:{TIKZ_SERVER_PORT}/renders/{name}.svg`

**错误处理：**
- 编译失败 → 返回 LaTeX 错误日志，方便 Claude 自行修正
- 工具链缺失 → 提示用户参考 setup.md 安装

### 2.3 Server 层

**tikz_server.py — 基于 Python 标准库的自定义 HTTPServer**

路由设计：
| 路径 | 方法 | 说明 |
|------|------|------|
| `/healthz` | GET | 返回 `{"status": "ok"}` |
| `/` | GET | 索引页，列出所有已渲染的 SVG（带缩略图） |
| `/renders/<name>.svg` | GET | 提供 SVG 文件 |
| `/renders/<name>.pdf` | GET | 提供 PDF 文件（下载用） |

**绑定与展示分离：**
- 绑定地址：硬编码 `0.0.0.0`（始终监听所有接口）
- 展示地址：由 `TIKZ_SERVER_HOST` 环境变量控制，用于拼接返回给用户的 URL

### 2.4 Server 生命周期管理

**ensure_server.py 流程：**

```
读取 ~/.cache/tikz-skill/server.json
  ↓
存在且 pid 存活？ ──否──→ 启动新 server
  ↓ 是
GET /healthz 成功？ ──否──→ kill 旧进程 → 启动新 server
  ↓ 是
返回当前 server 信息（复用）
```

**启动方式：**
```python
subprocess.Popen(
    [sys.executable, "tikz_server.py"],
    start_new_session=True,  # 脱离父进程
    stdout=log_file,
    stderr=log_file,
)
```
启动后轮询 `/healthz`（最多 5 秒），确认就绪后写入 `server.json`。

**停止方式：**
- `ensure_server.py --stop` → SIGTERM → 等待 3 秒 → 清理 server.json
- SKILL.md 中要求 Claude 在显式场景下使用 `--stop`（用户要求停止、端口冲突等）
- 正常使用不需要主动停止，server 跨会话复用

### 2.5 环境变量

| 变量名 | 用途 | 默认值 | 当前配置 |
|--------|------|--------|---------|
| `TIKZ_SERVER_HOST` | 展示 URL 的主机地址 | `127.0.0.1` | `10.8.0.2` |
| `TIKZ_SERVER_PORT` | 服务端口 | `8073` | `9094` |

配置位置：`~/.zshrc`
```bash
export TIKZ_SERVER_HOST="10.8.0.2"
export TIKZ_SERVER_PORT="9094"
```

## 3. SKILL.md 要点

**触发场景：**
- 用户要求画图、生成图表、可视化（TikZ 相关）
- 用户提到 TikZ、LaTeX 绘图、流程图、架构图等

**Claude 的工作流：**
1. 运行 `ensure_server.py` 确保 server 就绪
2. 编写 TikZ 代码
3. 调用 `render_tikz.py` 编译并获取 URL
4. 将 URL 返回给用户
5. 如果编译失败，读取错误日志并修正 TikZ 代码后重试
